# DESIGN.md - 樂譜移調工作台設計系統

## Design Summary (設計摘要)
本專案為一個極簡且高質感的音樂工具 Web App，專為處理吉他譜、五線譜的移調需求而設計。
介面採取「深色工作台模式 (Slate Dark Mode)」作為主軸，使用戶在編輯、上傳樂譜時能有沉浸、不刺眼的環境；同時，樂譜呈現區則採用「溫潤紙張模式 (Warm Ivory Paper)」，模擬真實樂譜的米色紙張質感，提供絕佳的看譜與列印體驗。

---

## Brand Voice (品牌調性)
- **專業而沉穩**：排除多餘的漸層色與霓虹燈效，使用純粹的深石板灰、靛藍與米白，營造音樂工作站（DAW）的專業氛圍。
- **實用至上**：資訊層級分明，按鈕、滑桿與控制項排列符合樂手在演出或練習時的操作直覺。
- **高易讀性**：五線譜的線條與音符大小合適，和弦譜的和弦符號與歌詞字級經過精心對齊，即使在小螢幕上也不易混淆。

## Typography (字體系統)
- **介面控制項** (UI Controls): `Outfit` / Sans-serif 字型，現代、大氣且在小字級下依然清晰。
- **樂譜與歌詞** (Sheet Music & Lyrics): `Lora` / Serif 字型，帶有溫暖優雅的襯線細節，適合長時間歌唱與樂譜閱讀。
- **記譜代碼** (ABC Notation): `Fira Code` / Courier New / Monospace 等寬字型，便於字元對齊與語法撰寫。

---

## Spacing and Layout (間距與佈局)
- **雙視窗對照模式** (Split Pane)：50% / 50% 左右對照。
  - 左視窗：PDF 閱讀器，背景色為深色。
  - 右視窗：樂譜工作台（包含編輯器與 Ivory 樂譜卡片）。
- **寬度與間距限制**：
  - 樂譜紙張卡片最大寬度限制為 `800px`，模擬實體 A4 比例，以保持舒適的視線移動範圍。

---

## Components and CTA (元件與互動)
- **移調控制項** (Transposer)：
  - 大而顯眼的 `-1` / `+1` 按鈕，點擊時立刻觸發移調渲染，字體動態縮放。
  - 移調顯示區（如 `+2 半音`）點擊即可重設回原調 `0`。
- **同步按鈕** (GAS Sync)：
  - 大色塊警告色橘黃按鈕，點擊有 Syncing 的 loading 動態。
- **PDF 文字擷取**：
  - 點擊後顯示 Loading 圖示，完成後以 Alert 提醒，順暢導入。

---

## Responsive Rules (響應式設計)
- **列印最佳化 (`@media print`)**：
  - 當用戶按下 `Ctrl + P` 或點選「列印」時，CSS 會自動隱藏所有介面元素（包括頂部工具列、GAS 同步條、PDF 預覽區、代碼編輯區）。
  - 將樂譜卡片鋪滿頁面寬度，去除陰影與圓角，背景轉為純白、文字轉為純黑，確保列印成 A4 紙張或另存成 PDF 時排版完美。
- **螢幕適應**：
  - 樂譜卡片中的和弦與歌詞採用 `word-group` 的 inline-flex 垂直排列，即使遇到手機窄螢幕，和弦與歌詞也會跟著「整組換行」，絕不會出現和弦與歌詞位置錯位的情況。

---

## Accessibility Rules (無障礙與易用性)
- 控制按鈕擁有足夠的對比度，且 Hover 時有明顯的背景色加深或外框亮起。
- 支援網頁字級動態調整（藉由頂部 `-` / `+` 按鈕），讓視力不佳的看譜者能放大字體。

---

## Implementation Notes (開發備忘錄)
1. **abcjs 整合**：
   - 由於 abcjs 會在 window/document 下操作 DOM，不宜使用 Next.js SSR 以免拋出執行期錯誤。故此專案選擇 Vite SPA 開發。
   - 移調功能可直接傳遞 `visualTranspose` 與 `midiTranspose` 參數，無需修改原始 ABC 語法，效能最佳。
2. **pdf.js 整合**：
   - 使用 `pdfjs-dist/build/pdf.worker.mjs?url` 本地引入 worker，維持 100% 離線可用。
   - 提取文字時，必須利用 Y 軸座標進行排序分組，並以 X 軸座標的間距推算並補上對應數量的空白，以完美重現和弦行的水平空格對齊。
